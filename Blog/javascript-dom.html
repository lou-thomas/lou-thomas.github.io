It's all about the flow, and sometimes a loop.

Control flow can be likened to reading a newspaper, generally speaking you will read from page 1, starting top left and working your way down the page. You just wouldn't read a sentence at the end of an article before reading the heading, intro and the meat of the article itself. Control flow is the same. It means code in your file is read and executed in order from the first line to the last. Mostly...dun, dun, dun.

Back to that newspaper, lets say its Sunday morning and you've read most of the news, in the order in which the Editor laid it out for you. You're having your coffee and now you're up to the good bit, the crosswords. You could look at the crosswords, read all the clues, but then if you don't stop the flow, you'll end up reading the answers on the next few pages before you filled out the crossword. This is where things like conditionals and loops are very important. They tell the program when to pause and pay attention to what the user is doing or check some other conditions before continuing on. In this case you might insert a loop. A loop is a handy way to instruct the program to do somthing multiple times. For example, look at clue 1, think, try to answer it, write your guess in the crossword grid. The loop knows you want to follow the same process for every clue in the puzzle, before carrying on to the other pages and the inevitable answers. You could even have a conditional either in the loop or somewhere else such as just before the page gets turned. In this case you could say, hold up control flow, don't be passing Go here untill this puzzle is complete and correct. Let's loop through each answer provided and see if it's right. IF it is, you may proceed. ELSE if it's not, you must ask grandma for help. A loop in this case means you don't have to write a new line of code to represent checking every clue in the puzzle. You only need to write a single loop which iterates through all the clues itself.
